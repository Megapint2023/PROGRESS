FINAL SCHOOL TEAM PROJECT:
DESCRIPTION:

This was a first team project i was a part of. Primary goal was to engineer a webpage for customer company office PC. Main goal was to automate the process of printing lables for shipping customer orders.

SOLUTION:

Since our customer already had a static company website we came up with a solution to not only automate printing but the entire prosess. We created and linked the website the the database and added webpages for registration, ordering and payment process. Then added separate database for company products, their prices and stock. Created a separate admin webpage for managing users, orders, product prices, stock and the lable printing. Additionally added advanced admin tools to modify and update any data such as product prices/stock/users/orderdetails etc. Also added automated calculators to track sales daily / weekly / monthly / yearly and some other admin tools. Printing process would this way function better for company and its users (since orders were made manually before by filling forms etc). This solution saved company lots of time and minimized the typo issues also.

For printing lables the data is retrieved with PHP and packed into PDF a file and structured with JavaScript to fit correctly for each lable size. Adminpage sends request to printer (via operating system) to print the data inside of PDF file. When adminpage "tries" to open PDF -> it will automatically "trigger" default PDF-viewer (or default API of the operating system) to open and read the PDF. At this point admin(user) has to confirm the action by clicking "print" button (so user interraction is required to finish the process). Information retrieved for the lables contains order id and information nesassary for the packaging and shipment.
